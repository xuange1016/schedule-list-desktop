# 桌面端任务管理应用 V1 页面原型与低保真布局说明

版本：V1.0  
日期：2026-05-15  
关联文档：docs/todoist-like-desktop-app-prd-v1.md  
用途：说明每个页面的大致布局、组件位置、弹窗结构和主要交互，帮助后续 AI 生成界面时不跑偏。

---

## 1. 这份文档是什么

页面原型说明和低保真布局文档，是介于“产品需求”和“正式设计稿”之间的说明文件。

它不追求颜色、字体、图标、动效等精细视觉，也不是最终 UI 图。它重点回答：

- 页面上有哪些区域
- 每个区域放什么组件
- 按钮大概在哪
- 用户点击后发生什么
- 弹窗里有哪些输入项
- 页面空状态长什么样

可以把它理解成“文字版草图”。后续真正开发界面时，AI 应先遵守这份低保真布局，再逐步做视觉美化。

## 2. 全局窗口结构

应用打开后直接进入任务管理主界面。

整体布局为左侧导航 + 右侧主内容区。任务详情不使用右侧固定面板，统一使用居中弹窗。

```text
--------------------------------------------------------------------------------+
|  左侧导航栏                    |  当前页面主内容区                             |
|-------------------------------|------------------------------------------------|
|  应用名称                     |  页面标题 / 日期 / 操作按钮                   |
|                               |------------------------------------------------|
|  收件箱        数量           |                                                |
|  今天          数量           |  任务列表 / 日期分组 / 筛选结果 / 日志记录    |
|  预览                         |                                                |
|  过滤                         |                                                |
|  日志                         |                                                |
|                               |                                                |
|  设置入口（预留）             |                                                |
+--------------------------------------------------------------------------------+
```

### 2.1 左侧导航栏

左侧导航栏固定显示，不随页面滚动。

组件：

- 应用名称
- 收件箱导航项
- 今天导航项
- 预览导航项
- 过滤导航项
- 日志导航项
- 设置入口预留

交互：

- 点击导航项切换页面
- 当前页面导航项高亮
- 收件箱、今天可显示未完成任务数量

视觉要求：

- 宽度适中
- 颜色克制
- 图标 + 中文文字
- 接近 Todoist 的紧凑导航风格

### 2.2 主内容区

主内容区根据当前页面变化。

通用结构：

- 顶部标题区
- 页面操作区
- 列表内容区
- 空状态

所有任务列表的任务行样式应保持一致。

## 3. 通用任务行原型

任务行是所有未完成任务页面的基础组件。

```text
+--------------------------------------------------------------------------------+
| ○  任务标题文字                                      日期  标签  P1  更多操作 |
|    任务描述摘要（可选，较浅色，最多一行）                                      |
+--------------------------------------------------------------------------------+
```

组件：

- 完成勾选按钮
- 任务标题
- 描述摘要
- 日期标识
- 标签标识
- 优先级标识
- 附件标识
- 提醒标识
- 更多操作按钮

交互：

- 点击圆形勾选按钮：任务完成，进入日志
- 点击任务标题或任务行主体：打开任务详情弹窗
- 点击更多操作：显示编辑、删除等快捷操作
- 鼠标悬停任务行：显示更多操作按钮

状态：

- 普通状态：显示任务基础信息
- 悬停状态：显示快捷操作
- 高优先级任务：优先级标识更明显
- 有附件任务：显示附件图标
- 有提醒任务：显示提醒图标

## 4. 任务详情弹窗原型

任务详情弹窗用于新建和编辑任务。

```text
                         +----------------------------------------+
                         |  新建任务 / 编辑任务              X   |
                         |----------------------------------------|
                         |  标题输入框                            |
                         |                                        |
                         |  详细描述输入框                        |
                         |                                        |
                         |  日期       [选择日期]                 |
                         |  优先级     [P1 v]                     |
                         |  标签       [学习] [生活] [工作] [+]   |
                         |  提醒       [选择提醒时间]             |
                         |  附件       [上传文件]  文件名列表     |
                         |----------------------------------------|
                         |  删除任务       取消        保存       |
                         +----------------------------------------+
```

### 4.1 新建任务弹窗

打开方式：

- 点击页面顶部“添加任务”
- 点击预览页面某一天的“添加任务”

默认值：

- 在收件箱中新建：日期默认为空
- 在今天页面新建：日期默认为今天
- 在预览页面某一天新建：日期默认为该日期
- 标签为空
- 优先级默认为 P4 或普通优先级

保存规则：

- 标题不能为空
- 点击保存后创建任务
- 点击取消后关闭弹窗，不保存

### 4.2 编辑任务弹窗

打开方式：

- 点击任务行主体
- 点击任务更多操作里的编辑

编辑项：

- 标题
- 详细描述
- 日期
- 优先级
- 标签
- 提醒
- 附件

保存规则：

- 点击保存后更新任务
- 点击取消后放弃未保存修改

### 4.3 附件上传

第一版需要支持真实上传文件。

弹窗内附件区域应包含：

- 上传文件按钮
- 已上传文件名列表
- 每个附件的打开入口

说明：

- 附件应与任务关联
- 附件文件名应在任务详情弹窗中可见
- 是否支持删除单个附件，开发时可作为增强项处理，但不允许批量删除

### 4.4 删除任务双重确认

删除入口在任务详情弹窗内。

流程：

1. 用户点击“删除任务”
2. 弹出第一次确认：“确定要删除这个任务吗？”
3. 用户确认后，弹出第二次确认：“删除后无法撤回，请再次确认”
4. 用户再次确认后，任务永久删除

规则：

- 只删除当前单个任务
- 删除后不可撤回
- 不提供撤销
- 不允许批量删除

## 5. 收件箱页面低保真布局

收件箱用于快速收集任务。

```text
+--------------------------------------------------------------------------------+
| 左侧导航栏 |  收件箱                                             + 添加任务   |
|-----------|--------------------------------------------------------------------|
| 收件箱    |                                                                    |
| 今天      |  ○  准备产品需求文档              今天   工作   P2                 |
| 预览      |     把功能范围整理清楚                                               |
| 过滤      |                                                                    |
| 日志      |  ○  临时想到的任务                  无日期  生活   P4                 |
|           |                                                                    |
|           |  + 添加任务                                                         |
+--------------------------------------------------------------------------------+
```

页面组件：

- 页面标题
- 添加任务按钮
- 未完成任务列表
- 任务行
- 空状态提示

交互：

- 点击添加任务，打开新建任务弹窗
- 点击任务，打开编辑任务弹窗
- 点击完成按钮，任务进入日志
- 点击删除入口，走双重确认流程

空状态：

```text
收件箱暂无任务
[添加任务]
```

## 6. 今天页面低保真布局

今天页面展示截止日期为今天的任务。

```text
+--------------------------------------------------------------------------------+
| 左侧导航栏 |  今天                         2026年5月15日 周五     + 添加任务 |
|-----------|--------------------------------------------------------------------|
| 收件箱    |  今日任务                                                          |
| 今天      |                                                                    |
| 预览      |  ○  完成应用原型说明                 工作   P1                     |
| 过滤      |  ○  整理任务标签规则                 学习   P3                     |
| 日志      |                                                                    |
+--------------------------------------------------------------------------------+
```

页面组件：

- 页面标题
- 今日日期
- 添加任务按钮
- 今日任务列表
- 空状态提示

交互：

- 只展示截止日期为今天的未完成任务
- 在今天页面新建任务时，默认日期为今天
- 任务可编辑、删除、完成

空状态：

```text
今天没有安排
[添加任务]
```

## 7. 预览页面低保真布局

预览页面按日期展示未来安排，每一天都显示。

```text
+--------------------------------------------------------------------------------+
| 左侧导航栏 |  预览                                                               |
|-----------|--------------------------------------------------------------------|
| 收件箱    |  5月15日 周五 · 今天                                + 添加任务      |
| 今天      |  ○  完成低保真文档                    工作   P1                     |
| 预览      |                                                                    |
| 过滤      |  5月16日 周六 · 明天                                + 添加任务      |
| 日志      |  暂无任务                                                          |
|           |                                                                    |
|           |  5月17日 周日                                      + 添加任务      |
|           |  ○  复盘第一版需求                    学习   P3                     |
|           |                                                                    |
|           |  5月18日 周一                                      + 添加任务      |
|           |  暂无任务                                                          |
+--------------------------------------------------------------------------------+
```

页面组件：

- 页面标题
- 日期分组
- 日期标题
- 每日添加任务按钮
- 当天任务列表
- 无任务提示

交互：

- 页面可向下滚动
- 每个日期都显示，即使没有任务
- 点击某一天的添加任务，弹窗默认日期为该日期
- 每天的任务都可以编辑、删除、完成

日期规则：

- 今天、明天显示特殊标识
- 日期从近到远排列
- 每天都保留添加入口

## 8. 过滤页面低保真布局

过滤页面用于按优先级或标签查看任务。

```text
+--------------------------------------------------------------------------------+
| 左侧导航栏 |  过滤                                                               |
|-----------|--------------------------------------------------------------------|
| 收件箱    |  优先级： [全部] [P1] [P2] [P3] [P4]                                |
| 今天      |  标签：   [全部] [学习] [生活] [工作] [+ 自定义标签]                |
| 预览      |                                                                    |
| 过滤      |  筛选结果                                                          |
| 日志      |  ○  完成产品文档                      工作   P1   今天              |
|           |  ○  复习前端组件                      学习   P3   5月17日           |
+--------------------------------------------------------------------------------+
```

页面组件：

- 页面标题
- 优先级筛选按钮组
- 标签筛选按钮组
- 清除筛选按钮
- 筛选结果列表

交互：

- 点击优先级后，只显示该优先级任务
- 点击标签后，只显示包含该标签的任务
- 可以清除筛选回到全部
- 筛选结果中的任务仍可编辑、删除、完成

标签规则：

- 默认标签：学习、生活、工作
- 用户可创建自定义标签
- 自定义标签出现在过滤页面标签列表中

## 9. 日志页面低保真布局

日志页面展示已完成任务，并支持恢复为未完成。

```text
+--------------------------------------------------------------------------------+
| 左侧导航栏 |  日志                                                               |
|-----------|--------------------------------------------------------------------|
| 收件箱    |  2026年5月15日                                                     |
| 今天      |  ✓  完成 PRD 文档                  原日期：今天   工作   P2  [恢复] |
| 预览      |  ✓  确认页面规则                   原日期：今天   学习   P3  [恢复] |
| 过滤      |                                                                    |
| 日志      |  2026年5月14日                                                     |
|           |  ✓  整理任务想法                   原日期：无     生活   P4  [恢复] |
+--------------------------------------------------------------------------------+
```

页面组件：

- 页面标题
- 完成日期分组
- 已完成任务行
- 原截止日期
- 完成时间
- 标签
- 优先级
- 恢复按钮

交互：

- 日志任务默认不可编辑
- 点击恢复按钮后，任务回到未完成状态
- 恢复后的任务重新出现在对应日期页面或筛选结果中

空状态：

```text
暂无已完成任务
完成任务后会显示在这里
```

## 10. 弹窗和确认框低保真说明

### 10.1 通用弹窗规则

- 弹窗居中显示
- 背景有轻微遮罩
- 点击右上角 X 或取消按钮关闭
- 表单底部放主要操作按钮
- 保存按钮放在右下角
- 删除任务按钮放在左下角，并使用危险色

### 10.2 删除确认框

第一次确认：

```text
+--------------------------------------+
| 删除任务                             |
|--------------------------------------|
| 确定要删除这个任务吗？               |
|                                      |
|          取消        继续删除        |
+--------------------------------------+
```

第二次确认：

```text
+--------------------------------------+
| 再次确认删除                         |
|--------------------------------------|
| 删除后无法撤回，请再次确认。         |
|                                      |
|          取消        确认永久删除    |
+--------------------------------------+
```

## 11. 页面之间的数据关系

同一个任务会根据状态和日期出现在不同页面。

```text
新建任务
  |
  +-- 未完成 + 无日期  -> 收件箱
  +-- 未完成 + 今天    -> 今天、预览、过滤
  +-- 未完成 + 未来日期 -> 预览、过滤
  +-- 已完成           -> 日志
```

说明：

- 收件箱偏向收集任务，但也可以显示未完成任务入口
- 今天页面只看今天截止任务
- 预览页面按日期看任务
- 过滤页面按条件看任务
- 日志页面只看已完成任务

## 12. 后续开发提醒

后续 AI 开发界面时，应优先遵守以下顺序：

1. 先保证页面结构和交互正确
2. 再做视觉接近 Todoist
3. 再做细节动效和体验优化
4. 不要为了好看加入与任务管理无关的大面积装饰
5. 不要把产品做成营销首页或展示页

第一版的重点是：

- 能创建任务
- 能编辑任务
- 能上传附件
- 能完成任务
- 能恢复日志任务
- 能按日期预览
- 能按优先级和标签过滤
- 能本地保存
