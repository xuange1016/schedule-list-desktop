# 桌面端任务管理应用 V1 用户流程与任务状态规则

版本：V1.0  
日期：2026-05-15  
关联文档：docs/todoist-like-desktop-app-prd-v1.md、docs/todoist-like-desktop-app-wireframes-v1.md  
用途：定义任务在不同页面、不同状态之间如何流转，减少开发阶段的理解偏差。

---

## 1. 这份文档解决什么问题

PRD 说明“要做哪些功能”，低保真文档说明“页面大概怎么摆”。本文件进一步说明：

- 任务从创建到完成要经过哪些状态
- 每个页面到底显示哪些任务
- 用户在不同页面新建任务时默认值是什么
- 完成、恢复、删除、上传附件等关键动作如何影响任务
- 哪些边界情况第一版必须处理

后续开发时，如果页面展示规则和本文件冲突，以本文件为准。

## 2. 核心产品裁决

### 2.1 收件箱的定义

V1 中，收件箱是所有未完成任务的总入口。

也就是说：

- 无日期任务显示在收件箱
- 今天任务显示在收件箱，也显示在今天页面
- 未来日期任务显示在收件箱，也显示在预览页面
- 已完成任务不显示在收件箱，只显示在日志

这样设计的原因：

- 第一版暂时没有“项目”功能
- 用户需要一个总览入口管理所有未完成任务
- 方便用户从一个页面完成新增、编辑、删除、完成等核心操作

### 2.2 页面不是任务归属地，而是任务视图

同一个任务可以同时出现在多个页面。

例如：

- 一个截止日期为今天的任务，会出现在收件箱、今天、预览、过滤结果中
- 一个未来任务，会出现在收件箱、预览、过滤结果中
- 一个无日期任务，会出现在收件箱、过滤结果中
- 一个已完成任务，只出现在日志中

用户修改任务时，所有页面应同步反映最新状态。

### 2.3 过滤页第一版不单独提供新增任务

过滤页面的主要作用是筛选和处理已有任务。

V1 中，过滤页面支持：

- 查看筛选结果
- 编辑任务
- 删除任务
- 完成任务

V1 中，过滤页面暂不提供独立新增任务按钮。

原因：

- 过滤页面的新增默认值容易产生歧义
- 用户可以在收件箱、今天、预览页面新增任务
- 第一版优先保证主要流程清晰

### 2.4 预览页展示范围

V1 中，预览页面默认展示从今天开始的未来 30 天。

规则：

- 每一天都显示日期分组
- 有任务的日期显示任务
- 没有任务的日期显示“暂无任务”和添加任务入口
- 后续版本可以增加加载更多日期、月份切换或日历视图

## 3. 任务状态模型

每个任务在 V1 中只需要关注以下状态。

### 3.1 基础状态

- 未完成
- 已完成
- 已删除

### 3.2 日期状态

- 无日期
- 今天
- 未来日期
- 过去日期

### 3.3 信息状态

- 有标题
- 有描述
- 有优先级
- 有标签
- 有提醒时间
- 有附件

### 3.4 状态组合说明

未完成任务可以：

- 没有日期
- 有今天日期
- 有未来日期
- 有过去日期

已完成任务：

- 从未完成列表移除
- 进入日志页面
- 保留原截止日期
- 记录完成时间
- 可以恢复为未完成

已删除任务：

- 从所有页面永久消失
- 不进入日志
- 不可恢复

## 4. 页面显示规则

### 4.1 收件箱

显示范围：

- 所有未完成任务

不显示：

- 已完成任务
- 已删除任务

排序建议：

1. 今天到期任务
2. 未来日期任务
3. 无日期任务
4. 过去未完成任务

说明：

- 排序只是第一版建议，实际视觉上可以先按创建时间或截止日期排序
- 关键是收件箱必须作为未完成任务总入口存在

### 4.2 今天

显示范围：

- 截止日期为今天的未完成任务

不显示：

- 无日期任务
- 未来日期任务
- 已完成任务
- 已删除任务

特殊情况：

- 如果某个任务原本日期为今天，完成后进入日志
- 如果日志恢复该任务，且原日期仍为今天，则重新显示在今天页面

### 4.3 预览

显示范围：

- 从今天开始未来 30 天的日期分组
- 每个日期分组中的未完成任务

不显示：

- 无日期任务
- 已完成任务
- 已删除任务

特殊情况：

- 没有任务的日期也必须显示
- 每个日期都有添加任务入口
- 今天和明天需要特殊标识

### 4.4 过滤

显示范围：

- 符合筛选条件的未完成任务

筛选条件：

- 优先级
- 标签

不显示：

- 已完成任务
- 已删除任务

特殊情况：

- 无日期任务也可以出现在过滤结果中
- 过滤结果中的任务可以编辑、删除、完成
- 过滤页面不单独新增任务

### 4.5 日志

显示范围：

- 所有已完成任务

不显示：

- 未完成任务
- 已删除任务

分组方式：

- 按完成日期分组
- 同一天内按完成时间倒序

操作：

- 查看完成记录
- 恢复为未完成

## 5. 新建任务流程

### 5.1 从收件箱新建任务

流程：

1. 用户点击收件箱顶部“添加任务”
2. 打开新建任务弹窗
3. 用户填写标题、描述、日期、优先级、标签、提醒、附件
4. 用户点击保存
5. 任务创建成功并显示在收件箱

默认值：

- 日期：空
- 优先级：P4
- 标签：空
- 提醒：空
- 附件：空

如果用户选择了今天日期：

- 任务显示在收件箱
- 同时显示在今天页面
- 同时显示在预览页面今天分组

### 5.2 从今天页面新建任务

流程：

1. 用户点击今天页面“添加任务”
2. 打开新建任务弹窗
3. 日期默认填入今天
4. 用户填写其他信息
5. 点击保存
6. 任务显示在今天页面，同时显示在收件箱和预览页面

默认值：

- 日期：今天
- 优先级：P4
- 标签：空

### 5.3 从预览页面某一天新建任务

流程：

1. 用户在某个日期分组点击“添加任务”
2. 打开新建任务弹窗
3. 日期默认填入该日期
4. 用户填写其他信息
5. 点击保存
6. 任务显示在该日期分组，同时显示在收件箱和过滤结果中

默认值：

- 日期：所点击的日期
- 优先级：P4
- 标签：空

## 6. 编辑任务流程

### 6.1 打开编辑

用户可以从以下页面打开任务详情弹窗：

- 收件箱
- 今天
- 预览
- 过滤

打开方式：

- 点击任务行主体
- 点击更多操作中的编辑

### 6.2 可编辑内容

用户可以编辑：

- 标题
- 详细描述
- 日期
- 优先级
- 标签
- 提醒时间
- 附件

### 6.3 保存后的页面同步

如果用户修改日期：

- 从今天改为未来日期：今天页面不再显示，预览对应日期显示
- 从未来日期改为今天：今天页面显示，预览今天分组显示
- 从有日期改为无日期：预览和今天不显示，收件箱仍显示

如果用户修改标签：

- 过滤结果应立即更新

如果用户修改优先级：

- 过滤结果应立即更新

---

## 7. 完成任务流程

### 7.1 勾选完成

流程：

1. 用户点击任务左侧完成按钮
2. 任务状态变为已完成
3. 记录完成时间
4. 任务从未完成页面移除
5. 任务进入日志页面

影响页面：

- 收件箱：移除
- 今天：如果原本显示，则移除
- 预览：如果原本显示，则移除
- 过滤：如果原本显示，则移除
- 日志：新增记录

### 7.2 完成任务不等于删除任务

完成任务保留完整数据。

保留内容：

- 标题
- 描述
- 原截止日期
- 标签
- 优先级
- 附件记录
- 提醒时间
- 创建时间
- 完成时间

---

## 8. 日志恢复流程

### 8.1 恢复为未完成

流程：

1. 用户进入日志页面
2. 点击某条已完成任务的“恢复”按钮
3. 任务状态改回未完成
4. 清空完成时间
5. 任务从日志移除
6. 任务重新进入对应未完成页面

恢复后显示规则：

- 有今天日期：显示在收件箱、今天、预览今天分组
- 有未来日期：显示在收件箱、预览对应日期
- 无日期：显示在收件箱
- 有标签或优先级：可出现在过滤结果中

### 8.2 恢复过去日期任务

如果任务原截止日期已经过去：

- 恢复后仍保留原截止日期
- 收件箱显示该任务
- 今天页面不显示
- 预览页面默认不显示过去日期

后续版本可以增加“逾期任务”页面或“过期未完成”分组。

---

## 9. 删除任务流程

### 9.1 删除入口

删除入口位于任务详情弹窗。

支持删除的页面来源：

- 收件箱
- 今天
- 预览
- 过滤

日志中的已完成任务第一版不提供删除入口，只提供恢复。

### 9.2 双重确认

流程：

1. 用户点击删除任务
2. 系统弹出第一次确认
3. 用户点击继续删除
4. 系统弹出第二次确认
5. 用户点击确认永久删除
6. 任务永久删除

规则：

- 删除只针对当前单个任务
- 删除后不可撤回
- 删除后不进入日志
- 删除后从所有页面消失

---

## 10. 附件上传流程

### 10.1 上传附件

流程：

1. 用户打开任务详情弹窗
2. 点击上传文件
3. 选择本地文件
4. 系统将文件关联到当前任务
5. 弹窗中显示附件文件名

显示规则：

- 有附件的任务行显示附件图标
- 任务详情弹窗显示附件列表
- 用户可以打开已上传附件

### 10.2 附件保存原则

V1 需要真正上传文件，而不是只记录路径。

产品规则：

- 附件属于任务的一部分
- 附件应能随任务本地保存
- 删除任务时，与该任务关联的附件记录也随任务失效
- 不做批量附件删除

### 10.3 同名附件

如果用户上传同名附件：

- 第一版允许保存
- 界面显示原始文件名
- 系统内部应避免文件冲突

这属于技术实现细节，用户界面只需要保证文件名可见。

## 11. 标签流程

### 11.1 默认标签

V1 默认提供：

- 学习
- 生活
- 工作

### 11.2 自定义标签

用户可以在任务详情弹窗中创建自定义标签。

规则：

- 自定义标签创建后，可以用于后续任务
- 自定义标签会出现在过滤页面
- 标签名称不能为空
- 标签名称不应重复

### 11.3 标签删除

V1 暂不做全局删除标签功能。

原因：

- 删除标签会影响已有任务
- 第一版优先保证创建和筛选
- 后续版本可增加标签管理页面

---

## 12. 优先级流程

### 12.1 优先级选项

V1 使用四级优先级：

- P1：最高优先级
- P2：较高优先级
- P3：普通优先级
- P4：低优先级

默认值：

- 新建任务默认 P4

### 12.2 优先级展示

任务行中显示优先级标识。

视觉建议：

- P1 最醒目
- P2 次之
- P3 普通
- P4 最弱

---

## 13. 提醒流程

### 13.1 V1 产品规则

提醒第一版先保存提醒时间。

是否触发系统通知，仍待后续确认。

### 13.2 展示规则

如果任务设置了提醒：

- 任务行显示提醒图标
- 任务详情弹窗显示提醒时间

---

## 14. 边界情况规则

### 14.1 标题为空

标题为空时不能保存。

提示文案：

“请输入任务标题”

### 14.2 日期为空

允许日期为空。

日期为空的任务：

- 显示在收件箱
- 可显示在过滤结果
- 不显示在今天
- 不显示在预览

### 14.3 标签为空

允许标签为空。

标签为空的任务：

- 可显示在收件箱、今天、预览
- 不会出现在某个标签筛选结果中

### 14.4 附件为空

允许附件为空。

### 14.5 过去日期任务

允许任务日期是过去日期。

V1 规则：

- 收件箱显示
- 今天不显示
- 预览默认不显示
- 过滤结果可显示

后续可增加逾期分组。

### 14.6 删除确认中取消

如果用户在任意一次删除确认中取消：

- 任务不删除
- 回到任务详情弹窗或当前页面

以下是 V1 第一轮开发必须完成的核心能力：

- 创建任务
- 编辑任务
- 删除任务双重确认
- 完成任务
- 日志恢复任务
- 上传附件
- 默认标签
- 自定义标签
- 优先级
- 日期选择
- 收件箱显示所有未完成任务
- 今天显示今天任务
- 预览显示未来 30 天每日分组
- 过滤按优先级和标签筛选
- 日志按完成日期分组
- 本地保存

## 17. 后续开发指挥顺序

从产品经理视角，下一步开发顺序建议如下：

1. 建立应用骨架和五个页面空壳
2. 实现任务数据模型和本地保存
3. 实现任务弹窗的新建与编辑
4. 实现收件箱、今天、预览页面的数据展示
5. 实现完成任务和日志页面
6. 实现日志恢复
7. 实现过滤页面
8. 实现附件上传
9. 实现删除任务双重确认
10. 做整体视觉打磨和可用性检查

开发期间，用户主要负责监督：

- 页面是否像任务管理应用
- 操作是否顺手
- 是否符合 Todoist 风格
- 是否有明显遗漏
