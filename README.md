# 日程清单

一款中文桌面端任务管理应用，灵感来自 Todoist，面向个人日程、任务规划和本地化轻量管理场景。

它不是云服务，也不依赖账号体系。任务、附件、头像和个人设置都会保存在用户自己的电脑本地，适合用于个人学习、工作安排，以及 AI 协同开发流程练习。

## 项目亮点

- 桌面端体验：基于 Electron 构建，可打包为 Windows 安装包。
- 中文优先：界面、文档和产品流程均以中文用户为默认对象。
- 本地保存：任务数据、附件和用户资料保存在本机用户数据目录。
- Todoist 风格：包含收件箱、今天、预览、过滤、日志等核心任务视图。
- 产品文档完整：项目包含需求说明、页面说明、低保真布局、用户流程和实施计划。
- 可持续迭代：当前是 V1 本地版，后续可继续扩展云同步、搜索、项目分组、日历视图等能力。

## 功能概览

### 收件箱

- 查看所有未完成任务
- 新增、编辑、删除任务
- 按今天、未来、无日期、逾期分组
- 支持任务标题、描述、日期、优先级、提醒、标签和附件

### 今天

- 只展示当天需要完成的任务
- 在今天页面新增任务时，默认日期为今天
- 支持和收件箱一致的任务编辑能力

### 预览

- 展示从当前日期开始的未来 30 天
- 每一天都会显示，即使当天没有任务
- 可直接给指定日期添加任务

### 过滤

- 按优先级筛选任务
- 按标签筛选任务
- 默认标签包含：学习、生活、工作
- 自定义标签会从任务中自动出现

### 日志

- 收集所有已完成任务
- 按完成日期分组
- 支持将已完成任务恢复为未完成状态

### 设置

- 自定义用户名
- 自定义个性签名
- 自定义头像
- 设置会保存到本地，重启后仍然保留

## 本地数据说明

本项目 V1 使用本地 JSON 文件保存数据，不需要数据库服务。

数据保存位置由 Electron 的 `app.getPath("userData")` 决定，通常位于 Windows 当前用户的应用数据目录下。不同电脑、不同系统用户之间的数据不会自动同步。

主要数据包括：

- 任务数据：`data/tasks.json`
- 用户资料：`data/profile.json`
- 附件文件：应用本地附件目录
- 头像文件：应用本地资料目录

## 普通用户如何使用

如果你只是想安装软件，不需要下载源码，也不需要安装 Node.js。

推荐分发方式是从 GitHub Releases 下载安装包：

```text
日程清单 Setup 0.1.0.exe
```

下载后双击安装即可。安装完成后，可以通过桌面快捷方式或开始菜单打开。

如果是开发者自己本地测试，也可以直接运行打包后的免安装版本：

```text
release/win-unpacked/日程清单.exe
```

注意：不要只单独移动这个 `.exe` 文件。`win-unpacked` 目录里还有运行所需的其他文件，建议创建快捷方式，而不是把 exe 单独拷走。

## 为什么不把安装包直接提交到仓库？

安装包更适合放在 **GitHub Releases**，而不是直接提交到源码仓库。

原因如下：

- 安装包体积较大，当前 Windows 安装包约 100 MB，直接提交会让仓库迅速变重。
- Git 不擅长管理二进制大文件，每次更新安装包都会增加仓库体积。
- 源码仓库应该主要保存代码、文档和配置，方便审查、协作和版本管理。
- 安装包属于“发布产物”，更适合通过 Releases 按版本分发。
- 用户下载软件时，去 Releases 页面拿安装包最清晰，也更符合开源项目习惯。

推荐做法：

```text
代码和文档：提交到 Git 仓库
安装包 exe：上传到 GitHub Releases
```

## 开发环境准备

需要先安装：

- Node.js
- npm
- Git

克隆项目：

```bash
git clone git@github.com:xuange1016/schedule-list-desktop.git
cd schedule-list-desktop
```

安装依赖：

```bash
npm install
```

启动开发环境：

```bash
npm run dev
```

开发模式会同时启动 Vite 和 Electron。

## 部署与发布方法

这个项目是桌面应用，不是传统网站，所以“部署”主要指发布安装包。

推荐流程：

1. 更新版本号
   - 修改 `package.json` 里的 `version`

2. 构建并打包

   ```bash
   npm run dist
   ```

3. 检查输出文件

   ```text
   release/日程清单 Setup x.x.x.exe
   ```

4. 在 GitHub 创建 Release
   - 打开仓库页面
   - 进入 Releases
   - 点击 Draft a new release
   - 填写版本号，例如 `v0.1.0`
   - 上传 `release/日程清单 Setup x.x.x.exe`
   - 发布 Release

5. 用户下载 Release 中的安装包并安装

## 开发协作说明

这个项目采用“产品文档先行”的 AI 协作流程：

2. 再生成产品需求说明书
3. 再生成页面需求、低保真布局和用户流程
